# Security Policy

## Reporting a Vulnerability

If you discover a security issue in DAVE Design System, please report it privately so it can be fixed before public disclosure.

Preferred reporting path:
- Use GitHub Security Advisories if available for this repository.

If GitHub Security Advisories are not available, open an issue and include the word `SECURITY` in the title. Do not post exploit details publicly.

## What to Include

Please include as much of the following as possible:
- affected package(s) or component(s) (for example `@dave/react`, `@dave/charts`, `@dave/tokens`, `@dave/docs`)
- repository branch or package version
- a clear description of the vulnerability and impact
- steps to reproduce the issue
- suggested mitigation or fix, if available
- any relevant code snippets or proof-of-concept details

## Response Policy

The maintainers aim to acknowledge valid reports within 7 business days. If you do not receive a response, please follow up after 14 days.

## Supported Scope

This project is a personal design system and documentation site. It is maintained as an experimental open-source project, and may not receive long-term support for older releases.

## Disclosure Policy

Please do not disclose security issues publicly until they have been fixed or an explicit release plan has been shared by the maintainers.

## Notes

This repository is MIT licensed, but it is provided as-is with no stability or support guarantee. Security reports are appreciated and help improve the project for everyone.
